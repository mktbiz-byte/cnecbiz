## 1. 시스템 개요

### 오픈클로란?

오픈클로(OpenClo)는 Instagram, YouTube, TikTok에서 크리에이터를 **자동으로 탐색**하고, **AI로 분석**한 뒤, **이메일 시퀀스를 발송**하여 CNEC 플랫폼에 가입시키는 시스템입니다.

### 전체 플로우

```
봇 크롤링 → 크리에이터 발견 → AI 분석 (점수/카테고리) → 승인/검토/제외
     ↓
  승인된 크리에이터 + 이메일 보유
     ↓
  1차 이메일 발송 → (3일 후) 2차 → (3일 후) 3차
     ↓
  회신 → 협업 진행 / 가입 → CNEC 가입 확인
     ↓
  매일 오후 6시 네이버웍스 리포트 자동 발송
```

### 리전 지원

| 리전 | 언어 | 이메일 템플릿 |
|------|------|--------------|
| 한국 (korea) | 한국어 | 한국어 |
| 일본 (japan) | 일본어 | 일본어 |
| 미국 (us) | 영어 | 영어 |

모든 페이지 우측 상단에서 리전 전환이 가능합니다. 각 리전별로 독립적으로 데이터가 관리됩니다.

## 2. 접속 방법

### URL
```
https://cnecbiz.com/admin/openclo
```

### 로그인
- cnecbiz.com 관리자 계정으로 로그인 필요
- `/admin/login` → Google OAuth 로그인
- `admin_users` 테이블에 등록된 이메일만 접근 가능

### IP 제한 (선택사항)
- `봇 설정 > IP 화이트리스트`에 IP를 등록하면, 해당 IP에서만 오픈클로 페이지 접근 가능
- IP가 하나도 등록되지 않으면 모든 IP에서 접근 허용
- 최초 설정 시에는 IP 등록 없이 접근 가능

## 3. 페이지별 기능 가이드

### 3.1 대시보드

**URL**: `/admin/openclo`

오픈클로의 전체 현황을 한눈에 볼 수 있는 메인 페이지입니다.

#### 화면 구성

| 영역 | 내용 |
|------|------|
| KPI 카드 4개 | 오늘 탐색 수, 신규 크리에이터, 이메일 발송, 가입 전환 |
| 봇 상태 | 가동중/지연/중단/미확인 (마지막 활동 기준 판단) |
| 가입자 수 | CNEC에 이미 가입한 크리에이터 수 |
| 플랫폼별 현황 | Instagram / YouTube / TikTok 각각의 크리에이터 수 |
| 최근 활동 | 봇의 최근 20건 활동 로그 (성공/실패 표시) |

#### 사용 방법

1. **리전 선택**: 우측 상단의 `한국 / 일본 / 미국` 버튼으로 전환
2. **네이버웍스 리포트 발송**: `네이버웍스 리포트 발송` 버튼 클릭 → 현재 화면의 KPI + 봇 상태 + 플랫폼 현황이 네이버웍스 메시지로 즉시 전송
3. **새로고침**: `새로고침` 버튼 또는 30초마다 자동 갱신

#### 봇 상태 판단 기준

| 상태 | 조건 |
|------|------|
| 가동중 (녹색) | 마지막 활동이 35분 이내 |
| 지연 (노란색) | 마지막 활동이 35~60분 사이 |
| 중단 (빨간색) | 마지막 활동이 60분 초과 |
| 미확인 (회색) | 활동 기록 없음 |

### 3.2 크리에이터 목록

**URL**: `/admin/openclo/creators`

봇이 발견했거나 수동 등록한 모든 크리에이터를 관리하는 페이지입니다.

#### 필터 옵션

| 필터 | 옵션 |
|------|------|
| AI 상태 | 전체 / 승인 / 검토 / 제외 / 미분석 |
| 가입 여부 | 전체 / 가입 / 미가입 |
| 컨택 상태 | 전체 / 미발송 / 1차발송 / 2차발송 / 3차발송 / 회신 / 가입완료 / 협업중 |
| 플랫폼 | 전체 / Instagram / YouTube / TikTok |
| 정렬 | 최신순 / 점수순 / 팔로워순 |
| 검색 | 유저네임 또는 이름으로 검색 |

#### 크리에이터 수동 등록 방법

1. 우측 상단 `크리에이터 수동 등록` 버튼 클릭
2. **프로필 URL 붙여넣기** (가장 편리한 방법)
   - Instagram: `https://instagram.com/username`
   - YouTube: `https://youtube.com/@channel`
   - TikTok: `https://tiktok.com/@username`
   - URL을 붙여넣으면 **플랫폼과 유저네임이 자동으로 감지**됩니다
3. 나머지 정보 입력 (선택사항)
   - 이름, 이메일, 팔로워 수, 바이오/메모
   - **이메일이 있어야 자동 이메일 발송 대상**에 포함됩니다
4. `등록` 버튼 클릭
5. 등록 후 AI 분석 실행 권장 (목록에서 해당 크리에이터 우측 `↻` 아이콘 클릭)

#### 일괄 작업

1. 체크박스로 여러 크리에이터 선택
2. 상단에 나타나는 액션 바에서:
   - `일괄 승인`: 선택한 크리에이터를 모두 승인 상태로 변경
   - `일괄 제외`: 선택한 크리에이터를 모두 제외 상태로 변경
   - `AI 재분석`: 선택한 크리에이터를 AI로 다시 분석

#### 상세 보기

- 크리에이터 행 클릭 → 상세 모달 열림
- 모달에서 확인 가능한 정보:
  - 기본 정보 (플랫폼, 팔로워, 바이오, 이메일 등)
  - AI 분석 요약
  - AI 분석 이력 (점수 변화 추적 가능)
  - 컨택 이력 (이메일 발송 기록)
- 모달에서 가능한 작업:
  - `AI 재분석` / `삭제` / `프로필 보기` (원본 SNS 프로필로 이동)

#### 의심 점수 해석

| 점수 범위 | 의미 | 색상 | 자동 처리 |
|-----------|------|------|----------|
| 0~30 | 양호 (진짜 크리에이터) | 초록 | → 자동 승인 |
| 31~70 | 검토 필요 | 노랑 | → 검토 대기열로 이동 |
| 71~100 | 의심 (가짜/봇 계정 등) | 빨강 | → 자동 제외 |

### 3.3 검토 대기열

**URL**: `/admin/openclo/review`

AI 분석 결과 수동 확인이 필요한 크리에이터(의심 점수 31~70)를 관리합니다.

#### 사용 방법

1. 의심 점수 순으로 정렬되어 표시됨 (낮은 점수 = 더 양호)
2. 각 크리에이터에 대해:
   - `승인`: 이메일 발송 대상에 포함
   - `제외`: 더 이상 연락하지 않음
   - `재분석`: AI에게 다시 분석 요청
3. `전체 승인` 버튼: 대기열 전체를 한 번에 승인

#### 표시 정보

- 플랫폼 + 유저네임 (클릭하면 원본 프로필 이동)
- CNEC 가입 여부 뱃지
- 팔로워 수, 카테고리
- AI 분석 요약
- 의심 점수 (큰 숫자로 표시)

### 3.4 봇 상태

**URL**: `/admin/openclo/bot-status`

봇의 실시간 상태와 활동 로그를 모니터링하고, 수동으로 봇을 실행할 수 있습니다.

#### 상태 요약 카드

| 카드 | 내용 |
|------|------|
| 봇 상태 | 가동중/지연/중단/미확인 |
| 오늘 활동 | 오늘 총 활동 건수 |
| 오늘 에러 | 오늘 발생한 에러 건수 |
| 미분석 대기 | AI 분석 대기 중인 크리에이터 수 |

#### 수동 트리거 (즉시 실행)

| 버튼 | 기능 |
|------|------|
| `Instagram 즉시 크롤링` | Instagram 봇 즉시 실행 → 키워드/해시태그 기반 크리에이터 탐색 |
| `YouTube 즉시 크롤링` | YouTube 봇 즉시 실행 → 채널 검색 및 정보 수집 |
| `TikTok 즉시 크롤링` | TikTok 봇 즉시 실행 → 유저 검색 및 정보 수집 |
| `AI 일괄 분석` | 미분석 크리에이터 최대 20명을 AI로 일괄 분석 |
| `네이버웍스 상태 발송` | 현재 봇 상태 + 에러 목록을 네이버웍스로 즉시 전송 |

#### 봇 수동 크롤링 실행 방법

1. 원하는 플랫폼의 `즉시 크롤링` 버튼 클릭
2. 봇이 해당 플랫폼에서 크리에이터를 탐색
3. 완료 시 팝업으로 결과 표시 (방문 수, 신규 발견 수)
4. 발견된 크리에이터는 `미분석(pending)` 상태로 등록
5. `AI 일괄 분석` 버튼으로 분석 실행 → 승인/검토/제외 자동 분류

#### 활동 로그

- 필터: 플랫폼별 / 성공-실패별
- 로그 내용: 시간, 플랫폼, 액션, 성공/실패, 소요시간, 에러 메시지
- 실패 행은 빨간색 배경으로 강조
- `새로고침` 버튼으로 최신 데이터 갱신

#### 네이버웍스 에러 리포트 발송

1. `네이버웍스 상태 발송` 버튼 클릭
2. 전송되는 내용:
   - 봇 상태 (가동중/중단 등)
   - 오늘 활동/에러 건수
   - 미분석 대기 수
   - 최근 에러 목록 (최대 10건, 시간/플랫폼/에러 내용)

### 3.5 봇 설정

**URL**: `/admin/openclo/bot-config`

플랫폼별 봇 설정, 이메일 템플릿, 네이버웍스 연동, IP 관리를 설정합니다.

#### 플랫폼별 봇 설정

상단 탭에서 Instagram / YouTube / TikTok 선택 후 설정:

| 설정 항목 | 설명 | 기본값 |
|-----------|------|--------|
| 활성/비활성 | 해당 플랫폼 봇 on/off | 활성 |
| 검색 키워드 | 봇이 검색할 키워드 목록 | 리전별 기본값 |
| 해시태그 | 봇이 탐색할 해시태그 목록 | 리전별 기본값 |
| 최소 팔로워 | 이 수 미만은 무시 | 1,000 |
| 최대 팔로워 | 이 수 초과는 무시 | 1,000,000 |
| 속도 모드 | 느림(안전) / 보통 / 빠름 | 보통 |
| 일일 제한 | 하루 최대 탐색 수 | 500 |

#### 키워드/해시태그 관리

1. `+ 추가` 버튼 클릭
2. 키워드 또는 해시태그 입력
3. 기존 태그의 `x` 버튼으로 삭제
4. **반드시 `설정 저장` 버튼**을 눌러야 적용됩니다

#### 이메일 템플릿 설정

| 항목 | 설명 |
|------|------|
| 이메일 발송 간격 (일) | 1차 → 2차, 2차 → 3차 사이 대기일 (기본: 3일) |
| 1차 이메일 템플릿 | 첫 번째 소개 이메일 (친절한 소개 톤) |
| 2차 이메일 템플릿 | 리마인더 이메일 (구체적 혜택 강조) |
| 3차 이메일 템플릿 | 마지막 기회 이메일 (특별 오퍼 제안) |

**사용 가능한 변수:**
- `{{creator_name}}` → 크리에이터 이름
- `{{platform}}` → 플랫폼명 (instagram, youtube, tiktok)
- `{{category}}` → 크리에이터 카테고리
- `{{followers}}` → 팔로워 수

> 참고: Gemini AI가 템플릿을 기반으로 각 크리에이터에게 맞춤 이메일을 생성합니다. AI 실패 시 원본 템플릿이 변수 치환되어 발송됩니다.

#### 네이버웍스 연동

- 웹훅 URL 입력 (선택사항, 기본적으로는 기존 환경변수의 네이버웍스 채널 사용)
- 리포트는 매일 오후 6시 KST에 자동 발송

#### IP 화이트리스트 관리

1. `내 IP 추가` 버튼: 현재 접속 중인 IP를 자동으로 추가
2. 수동 IP 추가: IP 주소 + 라벨(사무실 등) 입력 후 `추가`
3. 각 IP를 활성화/비활성화/삭제 가능
4. **IP가 하나도 없으면 모든 접근 허용** (초기 설정 편의)
5. IP 등록 시, 해당 IP에서만 `/admin/openclo/*` 접근 가능

### 3.6 이메일 관리

**URL**: `/admin/openclo/emails`

이메일 발송 퍼널, 대기 목록, 발송 이력, 회신 관리, 가입 전환을 관리합니다.

#### 이메일 발송 퍼널

화살표로 연결된 단계별 현황:

```
미발송 → 1차 발송 → 2차 발송 → 3차 발송 → 회신 → 가입
```

각 단계의 크리에이터 수가 표시됩니다.

#### 네이버웍스 이메일 현황 발송

- `네이버웍스 발송` 버튼 클릭 → 현재 퍼널 데이터 + 대기/회신/전환 현황이 네이버웍스로 즉시 전송

#### 발송 대기 목록

- 오늘 이메일 발송 가능한 크리에이터 목록 (최대 20명)
- 조건: 승인됨 + 미가입 + 이메일 있음 + contact_status가 none/email_1/email_2
- `전체 발송 실행` 버튼: 오늘 발송 예정인 이메일을 즉시 일괄 발송
  - 참고: 자동 발송은 매일 오전 10시 KST에 실행됨
  - 이 버튼은 자동 발송을 기다리지 않고 **즉시 수동 실행**할 때 사용

#### 발송 이력

- 기간 선택: 최근 7일 / 30일 / 90일
- 테이블 내용: 시간, 크리에이터, 이메일 주소, 회차(1/2/3차), 제목, 결과
- 결과 상태: sent(발송) / opened(오픈) / replied(회신) / bounced(반송) / registered(가입)

#### 회신 관리

- `회신` 상태인 크리에이터 목록
- `협업 진행` 버튼: 해당 크리에이터의 상태를 `collab`(협업중)으로 변경

#### 가입 전환 추적

- 이메일 발송 후 CNEC에 가입한 크리에이터 목록
- 표시 정보: 유저네임, 플랫폼, 전환까지 소요 일수
- 1차 이메일 발송일 → 가입일 기준으로 계산

### 자동 실행 스케줄

| 함수 | 실행 주기 | 설명 |
|------|----------|------|
| `scheduled-openclo-crawl` | 매 30분 | 활성화된 플랫폼에서 크리에이터 자동 크롤링 + AI 분석 |
| `scheduled-openclo-email` | 매일 10시 KST | 승인된 크리에이터에게 이메일 시퀀스 자동 발송 (최대 100건/일) |

| `scheduled-openclo-registration-check` | 매시간 | 크리에이터 이메일이 CNEC 가입자인지 확인 |

### 크롤링 자동 시간대별 플랫폼 배분

| KST 시간대 | 대상 플랫폼 |
|-----------|------------|
| 06:00 ~ 12:00 | Instagram |
| 12:00 ~ 18:00 | TikTok |
| 18:00 ~ 06:00 | YouTube |

> 해당 플랫폼이 비활성화 상태면 활성화된 다른 플랫폼으로 자동 전환

### 이메일 시퀀스 자동 흐름

```
1. 승인 + 이메일 보유 + 미가입 크리에이터 대상
2. contact_status = 'none' → 1차 이메일 발송
3. (3일 후) contact_status = 'email_1' → 2차 이메일 발송
4. (3일 후) contact_status = 'email_2' → 3차 이메일 발송
5. (7일 후) 무응답 → contact_status = 'no_response'로 변경
6. 중간에 가입 확인 시 → 발송 중단 + contact_status = 'registered'
```

### 자동 네이버웍스 알림

| 이벤트 | 알림 내용 |
|--------|----------|
| 매일 18시 KST | 일일 KPI 리포트 (전체 리전 합산 + 리전별 요약) |
| 월요일 18시 KST | 주간 요약 + TOP 5 신규 크리에이터 추가 |
| 크롤링 에러 발생 시 | 에러 리전/플랫폼/메시지 자동 알림 |
| 이메일 발송 완료 시 | 발송 성공/실패 건수 + 리전별 요약 |
| 치명적 에러 발생 시 | 에러 메시지 즉시 알림 |

## 8. FAQ / 트러블슈팅

### Q: 봇 상태가 계속 "중단"으로 표시됩니다
**A**: 봇은 30분마다 자동 실행됩니다. 최근 60분 이내 활동이 없으면 "중단"으로 표시됩니다.
- `봇 상태` 페이지에서 수동으로 `즉시 크롤링` 실행
- `봇 설정`에서 해당 플랫폼이 `활성` 상태인지 확인
- Netlify Functions 로그에서 에러 확인

### Q: 이메일이 발송되지 않습니다
**A**: 이메일 발송 조건을 확인하세요:
1. 크리에이터 `status`가 `approved`인가?
2. `is_registered`가 `false`인가? (이미 가입자면 발송 안 함)
3. `email` 필드에 이메일이 입력되어 있는가?
4. `contact_status`가 `none`, `email_1`, `email_2` 중 하나인가?
5. 발송 간격(기본 3일)이 지났는가?

### Q: AI 분석이 안 됩니다
**A**: `GEMINI_API_KEY` 환경변수가 설정되어 있는지 확인하세요. Netlify Functions 로그에서 에러 메시지를 확인하세요.

### Q: 네이버웍스 메시지가 안 옵니다
**A**: 기존 cnecbiz 네이버웍스 봇 설정이 정상인지 확인하세요:
- `NAVER_WORKS_CLIENT_ID`, `NAVER_WORKS_CLIENT_SECRET`, `NAVER_WORKS_BOT_ID`, `NAVER_WORKS_CHANNEL_ID` 환경변수 확인
- `/admin/openclo` 대시보드에서 `네이버웍스 리포트 발송` 버튼으로 수동 테스트

### Q: 수동으로 등록한 크리에이터가 이메일 발송 대상에 안 나옵니다
**A**: 아래 순서를 확인하세요:
1. 크리에이터 등록 시 **이메일 입력** 필수
2. 등록 후 **AI 분석 실행** (크리에이터 목록에서 `↻` 버튼)
3. AI 분석 결과 `status`가 **approved**가 되어야 함
4. 검토(review) 상태인 경우 `검토 대기열`에서 수동 승인 필요

### Q: 특정 크리에이터를 이메일 발송 대상에서 제외하고 싶습니다
**A**: 크리에이터 목록에서 해당 크리에이터를 선택 → `일괄 제외` 또는 개별 `삭제`
